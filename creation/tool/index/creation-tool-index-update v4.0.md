## Standard Artifact Template

### Universal Artifact Format
All artifacts follow this standardized format:

```markdown
# YYYYMMDDHHMM_descriptive_filename
[Store as: #tag/path/to/location vX.Y]
[Related: #tag/related/resource1, #tag/related/resource2]

---
title: "Document Title"
date: YYYY-MM-DD
type: essay|analysis|reference|template|etc.
status: draft|review|complete
tags:
  - relevant_tag1
  - relevant_tag2
---

# Document Title vX.Y

[Content formatted according to specific template type...]

## Version History

vX.Y - YYYY-MM-DD - Brief description of current version
vX.(Y-1) - YYYY-MM-DD - Brief description of previous version
v1.0 - YYYY-MM-DD - Initial version
```

## Standard Artifact Request Prompt

When requesting artifacts from Claude, use this standardized prompt structure:

```
Please create a [CONTENT TYPE] artifact with the following specifications:

Title: [TITLE]
Relevant Metadata: [ANY SPECIFIC METADATA NEEDED]
Key Content Requirements:
- [REQUIREMENT 1]
- [REQUIREMENT 2]
- [REQUIREMENT 3]

Structure:
- [SECTION 1]
- [SECTION 2]
- [SECTION 3]

Special Formatting Needs:
- [FORMATTING REQUIREMENT 1]
- [FORMATTING REQUIREMENT 2]

Cross-References to Include:
- [CROSS-REFERENCE 1]
- [CROSS-REFERENCE 2]

The artifact should follow our standard [CONTENT TYPE] template with proper formatting and cross-references. Please ensure it's complete and ready for direct use in Substack publication.
```

## Implementation in Claude Interactions

When working with Claude:

1. **Artifact Request**: Use standardized artifact request prompts
   ```
   Please create a [CONTENT TYPE] artifact following our standard template...
   ```

2. **Artifact Review**: Request review and validation of artifacts
   ```
   Please review this artifact for formatting issues and cross-reference integrity...
   ```

3. **Artifact Update**: Request specific updates to existing artifacts
   ```
   Please update this artifact by changing [OLD CONTENT] to [NEW CONTENT]...
   ```

4. **Artifact Validation**: Request validation against standards
   ```
   Please validate this artifact against our standards for [CONTENT TYPE]...
   ```

5. **Artifact Integration**: Request integration with other artifacts
   ```
   Please verify this artifact integrates properly with [RELATED ARTIFACTS]...
   ```
